01 — Getting Started

Opening the App

The Coding Fundamentals Trainer opens directly in any web browser. No installation is required to start using it — simply open index.html in your browser.

1
Open the App
Double-click index.html from your files folder, or open your browser and drag the file into it. The app loads with an animated splash screen.
💡 The splash screen shows for ~1.5 seconds and disappears automatically
2
View the Home Screen
The home screen (Landing Page) shows your stats (XP, puzzles solved, level, streak) and four module cards. Your progress is automatically loaded from your last session.
3
Navigate Using Bottom Bar
The bottom navigation bar gives quick access to: 🏠 Home, 🧩 Puzzles, 🔀 Logic, 🤖 Robot, 📊 Progress. Tap any icon to switch screens.
4
Start Learning
Tap any module card from the home screen, or tap the "🚀 Start Learning" button to go directly to Algorithm Puzzles — the recommended starting point.
02 — Installation

Installing as a Mobile App (PWA)

The app can be installed on Android phones and desktops to behave exactly like a native application — with its own icon, offline support, and full-screen mode.

1
Host the App Files
To install as a PWA, the files must be served from a local or online server. Options: VS Code Live Server (localhost), GitHub Pages (free online hosting), or Netlify (free drag-and-drop hosting).
2
On Android: Open in Chrome
Visit the hosted URL in Chrome on your Android device. After a few seconds, Chrome will show a banner: "Add Coding Fundamentals Trainer to Home Screen". Tap "Install".
💡 If banner doesn't appear: tap Chrome menu (⋮) → "Add to Home screen"
3
On Desktop: Install from Browser
In Chrome/Edge on desktop, look for the install icon (⊕) in the address bar. Click it and confirm. The app opens in its own window like a desktop application.
4
Use Fully Offline
After the first load, the Service Worker caches all files. Turn off your internet — the app continues to work completely. All features, all modules, all progress tracking.
✅ Tested offline on Android 12, Windows 11, and macOS

03 — Module 1
🧩
Algorithm Puzzles
Drag-and-drop sequencing • 3 difficulty levels • Instant feedback

Algorithm Puzzles teach students to arrange steps in the correct logical order — the foundational skill of algorithmic thinking. Real-world scenarios (making tea, sending a letter, computer startup) make abstract concepts concrete.

How to Use:

1
Select Difficulty Level: Choose Beginner (🟢), Intermediate (🟡), or Advanced (🔴) using the tabs at the top. Start with Beginner if new to the app.
2
Read the Question: Read the scenario carefully (e.g., "How do you make a cup of tea?"). Understand what the final goal is before arranging steps.
3
Drag and Drop Steps: On desktop: Click and drag each step card up or down to the correct position. On mobile: Long-press a step, then drag it. A number indicator shows its current position.
4
Check Your Order: When satisfied, tap "✅ Check Order". The app immediately evaluates your sequence and shows whether it is correct.
5
Read the Explanation: Whether correct or wrong, an explanation appears that explains the logical reasoning behind the correct order. READ THIS CAREFULLY — it contains the learning.
6
Try Again or Move On: Use "🔄 Reset" to reshuffle and try again with the same puzzle. Use "Next →" to advance to the next challenge.
✅ Pedagogical Tips for Teachers
→Use Beginner level as a class demonstration on a projector — ask students to vote on which step comes next.
→After a correct solve, ask students: "What would happen if Step 2 and Step 3 were swapped?" This develops dependency thinking.
→Connect to math: "An algorithm has a first step just like a proof has a hypothesis." Link coding to subject matter.
→The Algorithm Visual (at bottom of screen) shows the sequence in pseudocode — encourage students to copy it as notes.
⚠️ Common Mistakes to Avoid
Students often arrange steps in the order they find them on screen (top-to-bottom) without actually reasoning about dependencies. Remind students to ask: "What must happen BEFORE this step can happen?"
04 — Module 2
🔀
Logic Building
MCQ • Flowcharts • Pattern recognition • 3 levels

Logic Building develops conditional reasoning (IF-ELSE), loop tracing, compound logic, and pattern recognition — the building blocks of every decision-making structure in programming.

How to Use — Main Exercises:

1
Read the Logic Scenario: Each exercise presents a conditional statement or logic problem. For IF-ELSE questions, a flowchart diagram may appear below to visually represent the logic.
2
Evaluate Mentally First: Before tapping an answer, trace through the logic in your head. For IF-ELSE: first evaluate the condition (is it TRUE or FALSE?). For loops: trace each iteration step by step.
3
Select an Answer: Tap one of the four option buttons (A, B, C, D). The correct answer turns green, wrong answers turn red. All options are then disabled.
4
Read the Detailed Explanation: Whether correct or wrong, a full explanation appears that teaches the underlying concept. This is where the deep learning happens.
5
Move to Next Exercise: Tap "Next Exercise →" to advance. Exercises cycle through all questions at the selected difficulty level.

How to Use — Pattern Recognition Bonus:

1
Observe the Sequence: A number sequence is displayed (e.g., 2, 4, 8, 16, ?). Identify the rule or pattern connecting the numbers.
2
Calculate the Next Number: Apply the pattern to find the missing value. Type your answer in the input field.
3
Check and Advance: Tap "Check". Correct answers automatically advance to the next pattern after 2 seconds. Wrong answers prompt you to try again. Sequences include arithmetic, geometric (powers of 2), Fibonacci, and perfect squares.
✅ Pedagogical Tips for Teachers
→Use the flowchart display as a visual teaching tool. Pause on the flowchart slide and ask: "What happens if the condition is FALSE instead?"
→Connect pattern recognition to the maths curriculum — Fibonacci appears in nature, geometric sequences appear in finance and science.
→Intermediate level introduces compound conditions (AND/OR). This is ideal for Grade 8–10 students and directly prepares them for Boolean algebra.
→Advanced exercises on topological sorting can be used to introduce graph theory concepts informally.
05 — Module 3
🤖
Robot Maze — Problem Solving
Command programming • Grid navigation • 3 maze sizes

The Robot Maze module is a simulation where students program a robot by entering a sequence of movement commands. The robot executes the program step-by-step with visual animation. This develops planning, debugging, and optimization skills.

Understanding the Maze Grid:

SymbolMeaningDescription
🤖RobotYour robot's current position. Starts at top-left corner.
🏆GoalDestination the robot must reach. Always at bottom-right area.
🧱WallBlocked cell. Robot cannot move into walls.
EmptyPathFree cell the robot can move through.

How to Program the Robot:

1
Analyze the Maze: Before adding any commands, study the maze. Trace a valid path from 🤖 to 🏆 visually. Count how many steps UP, DOWN, LEFT, RIGHT are needed.
2
Add Commands: Tap the command buttons: ⬆️ UP, ⬇️ DOWN, ⬅️ LEFT, ➡️ RIGHT. Each tap adds one move to the Command Queue. Maximum 20 commands.
3
Review the Queue: The Command Queue shows all your programmed moves as colored chips. Click any chip to remove it. Plan carefully — like a real programmer!
4
Run the Program: Tap "▶ Run Program". Watch the robot animate through each command one by one (300ms per step). If it hits a wall, it stops early.
5
Interpret Results: If the robot reaches 🏆: SUCCESS! You see how many commands it took. If not: analyze where it stopped and debug your command sequence. Tap Reset to try again.
6
Optimize: After a successful run, challenge yourself to solve it in fewer commands. Fewer moves = more efficient algorithm. This introduces computational efficiency thinking.
✅ Pedagogical Tips for Teachers
→Unplugged first: Before using the app, do a physical maze activity. Draw a 5×5 grid on the blackboard, have a student act as the robot, and take verbal commands from the class.
→After solving, ask: "Can anyone find a shorter path?" This introduces optimization — a core computer science concept.
→Connect to real-world: GPS apps find shortest paths using Dijkstra's algorithm. Self-driving cars program movement commands. Warehouse robots at Amazon use exactly this logic.
→The debugging cycle (run → observe failure → fix → rerun) is identical to real-world programming. Name it explicitly: "This is what programmers call debugging!"
→Hard level (7×7 maze) is suitable for competitive activity — which team programs the shortest path wins.
⚠️ Common Mistake
Students often tap commands randomly without planning first. Emphasize: a programmer plans before they type. Require students to write their command sequence on paper before entering it into the app.
06 — Module 4
📦
Concept Blocks
Visual concepts • Live demos • Block builder

Concept Blocks introduces the four core programming concepts — Sequence, Loop, Condition, Function — through interactive explanations, animated demos, and a visual block builder. No syntax required.

The Four Concept Panels:

ConceptReal-World AnalogyProgramming Use
1️⃣ SequenceGetting ready for school — steps in orderEvery program runs line by line in sequence
🔁 LoopAlarm clock repeating every dayFOR loops, WHILE loops — repeat code
❓ ConditionTraffic light — stop if red, go if greenIF-ELSE statements — program decisions
⚙️ FunctionRecipe — write once, cook many timesFunctions — reusable code blocks

How to Use Concept Panels:

1
Open a Concept: Tap any concept panel header to expand it. The panel reveals a detailed explanation with real-world analogies and a visual code block showing the concept.
2
Read the Code Block: The colored code block shows pseudocode (plain English structured like code). This bridges the gap between everyday language and programming syntax.
3
Run the Live Demo: Each concept has a "🎮 Try Demo" button. Click it to see an animated simulation of the concept executing in real-time. For loops: watch each iteration appear. For conditions: see the branch taken based on a random value.
4
Study the Flowchart: The Condition concept includes an interactive flowchart showing branching paths. This visual representation is the standard way programmers design logic before coding.

How to Use the Block Builder:

1
Add Blocks: Click the block buttons: 🟢 START, ➡️ MOVE, ❓ IF, 🔁 LOOP, 🔴 END. Each block represents a programming construct.
2
Build a Program: Create a valid program sequence. A complete program typically starts with START and ends with END. Add MOVE, IF, and LOOP blocks in between to create logic.
3
Run the Program: Tap "▶ Run". The output panel shows a step-by-step execution trace — exactly what a real programming language does when it runs your code.
✅ Pedagogical Tips for Teachers
→Use Concept Blocks as a pre-lesson activator. Show the Sequence demo before a Maths lesson on ordered operations.
→The loop demo is perfect for introducing multiplication tables: "LOOP 5 times: add 3" = the 3-times table.
→After the Condition lesson, give students a simple IF-ELSE scenario from daily life and ask them to draw their own flowchart.
→The Function concept connects to mathematics: a function in maths (f(x) = 2x) is exactly the same idea — a rule applied to any input.

10 — FAQ

Frequently Asked Questions

Does the app need internet to work? ▼
No. After the first load (which requires internet for Google Fonts), the Service Worker caches everything. The app works fully offline. If fonts cannot load offline, the app falls back to system fonts automatically — all functionality remains intact.
Will my progress be saved if I close the browser? ▼
Yes. All progress is saved to LocalStorage automatically after every correct answer. It persists across browser sessions, app restarts, and even device restarts — as long as the same browser is used and browser data is not cleared.
Can multiple students share one device? ▼
The app stores one profile per browser. Multiple students can use the same device by: (1) using different browser profiles, or (2) using the "Reset Progress" button between students. A future version may include multi-profile support.
What age/grade is this app for? ▼
The app is designed for Grade 5 through Grade 12. Beginner levels are suitable for Grade 5–7, Intermediate for Grade 7–10, and Advanced for Grade 9–12. The content adapts through the difficulty system, making it appropriate for a wide range of learners.
The drag-and-drop doesn't work on my phone. What should I do? ▼
On some older Android devices, drag-and-drop may be less responsive. Try: long-pressing the step for 1 second before dragging. If still unresponsive, use a mouse/stylus. The app is also being updated to include touch-optimized swipe controls.
How do I convert this app into an Android APK? ▼
Host the app on GitHub Pages or Netlify, then visit PWABuilder.com, enter your URL, and click "Build → Android". PWABuilder generates an APK/AAB file you can distribute or upload to the Google Play Store. This process takes approximately 10 minutes.
11 — Deployment

Hosting & Deployment

🌐
GitHub Pages (Recommended — Free)
Create a free GitHub account → New repository → Upload all app files → Settings → Pages → Select main branch → Deploy. Your app is live at username.github.io/repo-name
⚡
Netlify (Easiest — Free)
Visit netlify.com → Drag your entire project folder onto the deploy area → Live URL generated instantly. No account required for basic deployment. Custom domain available.
📱
Android APK via PWABuilder
Host app online → Visit pwabuilder.com → Enter URL → Build Android APK → Download and install on devices via ADB or share as APK file. No Google Play account needed for direct distribution.
🏫
School Intranet Distribution
Place all files on a school server accessible via local network. Students connect to school WiFi and access 192.168.x.x:8080/index.html. After first visit, Service Worker enables offline use even after disconnecting from WiFi.
